Lignes de commandes GitHub

Tout d'abord, il ne faut pas oublier que le site de GitHub offre une explication à toutes ses commandes. Pour cela, on utilisera les commandes suivantes, qui nous rameneront directement sur les pages web GitHub concernées.

git help config
git help push
git help pull
git help branch

L’une des commandes git les plus utilisées est git config. On l’utilise pour configurer les préférences de l’utilisateur : son mail, l’algorithme utilisé pour diff, le nom d’utilisateur et le format de fichier etc.

git config

Ajouter un fichier:

git add "nom fichier"

Pour ajouter un fichier on utilisera:

git commit -m "message"

Pour cloner un depot:

git clone "repertoire" -b "branche"

Pour créer la clé SSH:

ssh-keygen -t rsa -b 4096 -C "mail@example.com"

Pour push, c'est-à-dire rendre disponible les fichiers sur lesquelles on travaille.

git push

Pour pull, c'est à dire recuperer les fichiers qui ont été push par les autres.

git pull

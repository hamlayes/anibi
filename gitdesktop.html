    • 0. Connexion à GitHub

    Avant de commencer, il est important de faire le liens entre notre repository locaux et ceux en ligne. Il faut donc une connexion entre GitHub Desktop et GitHub. Pour se faire il faut aller dans les paramètre de Github Desktop en allant dans File => Options => Account

    On peut ensuite faire Sign-in et un lien nous emmenera directement sur Internet où nous devrons nous connecter à notre compte GitHub.



    1. Repository en cours

    Ceci permet de connaître sur quel projet nous sommes entrain de travailler. On peut ajouter plusieurs projets en simultané utilisant Git et naviguer entre chaquer via ce menu.


    On peut y créer directement un repository, en cloner un depuis github directement ou encore ajouter git à un projet déjà existant depuis le bouton "ADD".


    2. Changement effectué sur les fichiers

    On voit ici tout les fichiers qui ont subit une modification depuis le dernier Commit. Toutes les modifications s'afficherons ici après un CTRL+S.

    On peut séléctionner les fichiers sur lesquels nous ferons notre Commit.


    3. Ajouter le commit (nom / description)

    Cette partie permet de mettre à jour l'historique du projet. On peut lui donner un nom et une description.


    Il est important de mettre des noms pertinant pour pouvoir differencier les modifications, les ajouts de fonctionnalité...


    History

    History est l'endroit où on peut retrouver tout les anciens Commit effectuer pour voir les changements effectués


    4. Branche en cours et création de branches


    Les branches sont des copies du programme principal où l'on peut soit ajouté des fonctionnalités pour la fusionnée avec la branche principal plus tard, soit l'utiliser pour faire des tests sans impacter le code principal. Il permet aussi de travailler à plusieurs. Comme sur l'image, on peut créer une branche pour chaque personne qui travaile sur le projet. De ce fait, tout le monde peut travailler sur sa partie sans risquer d'impacter les autres.

    /!\ Si une personne push son programme sur GitHub, les modifications qu'il n'a pas mais qui sont sur GitHub seront écrasé /!\


    5. Fetch et Push


    Ce sont les boutons qui permettent de lié GitHub Desktop à GitHub. Lorsque le compte est connécté. Ce bouton permet d'envoyer et de recevoir les modifications. Le bouton PUSH s'affiche quand une modification à été apporté et non envoyé sur GitHub. A l'inverse, quand le bouton FETCH est affiché, on peut recevoir la dernière version du programme

    /!\ Les données envoyées / recus dépendent de la branche sur la quelle vous êtes. Si vous êtes sur la branche 1, vous enverer les données sur la branche 1. Donc si quelqu'un d'autre travail sur la branche 2, il ne pourra pas recuperer les données juste envoyé. Il faudra qu'il change de branche. /!\


    6. Visualisation des changements sur les fichiers


    Cette partie permet de voir les modifications qui ont été apporté sur un fichier. En vert s'affiche les choses qui ont été ajouté, en rouge, les choses supprimés. On peut cirucle rentre les différents fichiers modifié sur la gauche de cette partie.
